## ðŸ“… Daily Routine (2-3 hours minimum)

### Morning (30 minutes)
1. **GitHub Contribution** (15 min)
   - Make at least 1 commit to any repository
   - Can be: bug fix, feature addition, documentation update, or learning project
   - **Why:** Shows consistent activity and dedication

2. **LeetCode Problem** (15 min)
   - Solve 1-2 LeetCode problems
   - Focus on Medium difficulty (FAANG standard)
   - Document your solution approach
   - **Why:** Technical interviews heavily focus on algorithms

### Afternoon (1-2 hours)
3. **Project Work** (1 hour)
   - Work on a portfolio project
   - Add features, fix bugs, or refactor code
   - Write clean, production-ready code
   - **Why:** Demonstrates real-world coding ability

4. **System Design Study** (30 min)
   - Study one system design concept
   - Design a system on paper (e.g., "Design Twitter", "Design Uber")
   - Review existing system designs
   - **Why:** FAANG interviews always include system design

### Evening (30 minutes)
5. **Documentation & Learning** (15 min)
   - Update README with progress
   - Write technical blog post (weekly)
   - Review what you learned today
   - **Why:** Shows communication skills and continuous learning

6. **Networking** (15 min)
   - Connect with FAANG engineers on LinkedIn
   - Comment on tech posts
   - Share your learning journey
   - **Why:** Referrals are the best way to get interviews
